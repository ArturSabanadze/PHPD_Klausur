Es soll eine Frontend/Backend Applikation entwickelt werden. Die Website soll mit HTML, CSS, PHP, OOP, AJAX, MVC Entwurfsmuster, MYSQL und PDO Webtechnologien realisiert werden. (Optional: auch die Webtechnologien JavaScript und jQuery)
Thema:______________________ Film Portal Applikation _______________________
Anforderungen:
0. README Datei
    1. einige Partial-Dateien.
    2. einige Template-Dateien.
3. mindestens zwei CSS-Dateien.
    3. Einige Bilder und Grafiken.
    4. Alle Seiten-Navigationen als Partials.
5. Der Controller soll alle Aktionen Kontrollieren.
6. Ein Webseite für die Dokumentation(HTML,CSS) in der Admin-Bereich sichtbar.
7. Alle Quellcode Dateien (PHP, MYSQL) müssen kommentiert werden.
8. Eine Dokumentation mit 2 UML Bilder und „Screen shots“ in ‚HTML‘-Format.
    9. Die Website soll Benutzerfreundlich sein.
    10. Es soll eine Backend/Frontend Website sein.
    11. Die Website soll sicher, was die Eingaben der Benutzer betrifft, sein.
    12. Die Dateien sollten in entsprechende Unterordner platziert werden.
    13. Die fertige Website soll in die Ordner „Klausurprojekt“ kopiert werden.
    14. Fazit – Erfahrungen, Erfolge und Misserfolge.
    Die HTML Seiten müssen die folgenden Bedingungen erfüllen: <html>, <head>, <body> ,<title> und die wichtigsten meta-Tags ein Link zu externe Stylesheet-Dateien, wenn nicht ‚embedded‘

Zusätzlichen Seiten:
    Eine Start-, Impressum-, Kontakt-, About- und AGB-Seite

Zur PHP, MySQL und PDO sollen die Skripte die folgenden Bedingungen erfüllen:
    1. Ein SQL-Skript für die Herstellung der Datenbank und einige Migrationsskripte.
    2. Ein Such-Formular - [auto-suggest] mit AJAX.
    3. Ein Login/Logout-Formular.
    4. Ein Anmeldungsformular (Registrierung).
    5. Ein Kontaktformular.
    6. Eine Listen-Sicht und Einzel–Sicht Design in Templates.
    7. Backend Für die Administration mit Login/Passwort.
    8. Der Zugang zur Backend soll nicht in der Navigation sichtbar sein.(separate URL)
    9. Angemeldeten dürfen Kommentare zu Filme schreiben.
10. Angemeldeten dürfen Filmdaten in JSON, XML aus OMDB-API holen.
11. Angemeldeten dürfen Filmdaten in JSON, XML aus Dateien holen.
    12. Angemeldeten dürfen eine Nachricht senden - Kontaktformular.
    13. Der Admin darf Kommentare löschen und/oder User & deren Kommentare löschen.
- 14. Der Admin kann Kontaktdaten lesen.
15. Der Admin kann Datenbank Backup-Skripte ausführen.
16. Der Admin kann Filmdaten exportieren in 3 Daten-Formate(XML, CSV, JSON)
    17. Der Admin kann Filmdaten importieren von OMDB.API oder von Dateien (3-Formate)
18. Mind. 1 trait, mind. 1 interface und mind. 1 abstract class erstellen.
19. Type-Hinting und Instanceof Operator sollten implementiert werden.
    20. Formulare Filtern/Validieren.
    21. Fehlermeldungen und/oder Erfolgsmeldungen.
    22. Backend mit eigenem Controller bauen.
    23. Class Funktionen sollten eingesetzt werden.
    24. Pagenation sollte implementiert werden.
25. [Statistiken] zur Filme, Schauspieler, Genre, … und User
